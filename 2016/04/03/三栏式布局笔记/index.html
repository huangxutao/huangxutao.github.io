任务核心

  • 实现两边固定宽，中间自适应
  • 最外层父容器的高度取决于三个子元素中最高的高度

实现过程

方案一 (最初方案)
  • 实现技巧，两边侧栏(.left, .right)分别左右浮动，中间(.main)采用postion: absolute,父容器(.content)position: relative

  • DOM结构:

  • CSS样式

  • demo1结果如图，显然没有达到预期效果，.content的高度无法自适应.main；究其原因，absolute 使.main脱离了文档流，即.main不会占据.content的空间

方案二
  • 实现技巧，两边侧栏(.left, .right)分别左右浮动，中间(.main)设置margin: 0 <.right的宽度+.content的内边距> 0 <.left的宽度+.content的内边距>

  • DOM结构

  • CSS样式

  • demo2结果如图，恩挺好需求都达到了，但是最后知道真相的我T_T，这并不是最优决绝方案

  • 存在的问题，当一个真实的页面中.main一般会放些相对重要的东西，所以要优先渲染，而该方案中.main会在.left, .right之后渲染，恩好了，所以就有了方案三和方案四
方案三 (圣杯布局)
  • 实现技巧，
    • 首先，三栏全部浮动，中间(.main)宽度为100%，左右两栏宽度固定
    • 然后，.left设置margin-left: -100%;，.right设置margin-left: - <.right的宽度>;使三栏并排显示
    • 此时，左右两栏是会把中间栏遮盖掉部分的，所以要给外层父容器.content设置内边距将中间栏挤出来
    • 最后，左右两栏(.left, .right)设置position: relative; left: -...;，大小为其宽度的负值（可根据实际情况是否加上边距）

方案四 (双飞翼布局源于淘宝UED)
  • 实现技巧 (前一半的实现与圣杯差不多)
    • 首先，三栏全部浮动，中间(.main)宽度为100%，左右两栏宽度固定
    • 然后，.left设置margin-left: -100%;，.right设置margin-left: - <.right的宽度>;使三栏并排显示
    • 此时，与圣杯布局一样，左右两栏是会把中间栏遮盖掉部分的，当然解决办法就不同于的圣杯了
    • 除了给外围父容器设置内边距外，还可以考虑在.main中增加一个容器.box
    • 最后给.box设置相应的margin值
